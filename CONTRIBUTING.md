# Contributing to DocuMind AI

Thank you for your interest in contributing to DocuMind AI! We welcome contributions that improve features, performance, code cleanliness, or documentation.

## How to Contribute

1. **Fork the Repository**: Create a personal copy of the repository on GitHub.
2. **Clone Locally**: `git clone https://github.com/username/documind-ai.git`
3. **Set Up the Environment**:
   - Create a virtual environment: `python -m venv .venv`
   - Activate it: `.venv\Scripts\activate` (Windows) or `source .venv/bin/activate` (macOS/Linux)
   - Install dev dependencies: `pip install -r requirements.txt`
4. **Create a Feature Branch**: `git checkout -b feature/amazing-feature`
5. **Code Style**:
   - Write clean, modular, object-oriented code.
   - Use PEP 8 styling conventions.
   - Ensure all functions have type hints and descriptive docstrings.
6. **Write and Run Tests**:
   - Verify changes with pytest: `pytest tests/ --cov=src`
   - Ensure you maintain or exceed the 80%+ test coverage.
7. **Commit and Push**: Write descriptive commit messages.
8. **Open a Pull Request**: Provide a clear explanation of your changes, what problem they solve, and test validation.

Thank you for making DocuMind AI better!
